\chapter*{Annexe : Documentation Générale et Manuel d'Utilisation}
\addcontentsline{toc}{chapter}{Annexe : Documentation Générale et Manuel d'Utilisation}
\label{annexe:doc}

Cette section fournit les directives nécessaires à la mise en œuvre, à la maintenance et à l'utilisation du système CNC 5 axes conçu dans le cadre de ce projet.

\section{Guide d'Utilisation (Manuel Opérateur)}

\subsection{Configuration Initiale et Branchements}
\begin{enumerate}
    \item \textbf{Alimentation :} S'assurer que le bouton d'arrêt d'urgence (E-STOP) est relâché. Brancher le câble secteur $220\text{V}$.
    \item \textbf{Connexion Wi-Fi :} Allumer la machine. L'ESP32 crée un point d'accès nommé \texttt{CNC\_5AXIS\_AP}. Se connecter avec le mot de passe par défaut.
    \item \textbf{Lancement de l'App :} Ouvrir l'application Flutter sur la tablette ou le PC. L'icône de statut Wi-Fi en haut à droite doit passer au vert.
\end{enumerate}

\subsection{Procédure de Mise à l'Origine (Homing)}
Avant tout usinage, la machine doit calibrer ses axes :
\begin{itemize}
    \item Appuyer sur le bouton \textbf{Homing} ($\$H$). Les axes Z, X, et Y se déplaceront successivement vers leurs capteurs de fin de course.
    \item Les axes rotatifs A et B doivent être alignés manuellement si les encodeurs absolus ne sont pas présents.
\end{itemize}

\subsection{Chargement et Exécution du G-Code}
\begin{enumerate}
    \item Utiliser le bouton \textbf{"Importer G-Code"} pour sélectionner un fichier `.nc` généré par votre logiciel de FAO (ex: Fusion 360 avec post-processeur adapté).
    \item Vérifier les limites de la pièce sur la visualisation 3D de l'application.
    \item Appuyer sur \textbf{Play} pour démarrer l'usinage.
\end{enumerate}

\section{Documentation Technique (Développeur)}

\subsection{Architecture du Micrologiciel (ESP32)}
Le code est structuré selon un modèle asynchrone utilisant FreeRTOS :
\begin{itemize}
    \item \textbf{Tâche de Communication (Core 0) :} Reçoit les trames JSON UDP et gère le parsing.
    \item \textbf{Tâche de Planification (Core 1) :} Exécute les algorithmes de cinématique inverse en temps réel.
\end{itemize}

\subsection{Protocole de Communication JSON}
L'application et le firmware communiquent via des trames JSON minimalistes pour optimiser la latence :
\begin{itemize}
    \item \textit{Exemple d'envoi :} \texttt{\{"c":"G1", "x":10.5, "f":500\}}
    \item \textit{Exemple de réponse :} \texttt{\{"s":"IDLE", "x":0.00, "y":0.00, "z":0.00\}}
\end{itemize}

\section{Maintenance et Sécurité}

\subsection{Entretien Mécanique}
\begin{itemize}
    \item \textbf{Lubrification :} Appliquer de la graisse lithium sur les vis à billes toutes les 50 heures d'utilisation.
    \item \textbf{Tension des courroies :} Vérifier la tension des courroies GT2 de l'axe A pour éviter les jeux de renversement.
\end{itemize}

\subsection{Consignes de Sécurité}
\begin{itemize}
    \item \textbf{Protection Oculaire :} Le port de lunettes de sécurité est obligatoire lors de l'usinage.
    \item \textbf{E-STOP :} Toujours garder une main à proximité du bouton rouge d'arrêt d'urgence.
    \item \textbf{Refroidissement :} Ne pas obstruer les ventilateurs du coffret électronique pour éviter la surchauffe des drivers DM556.
\end{itemize}
